Flask Application Overview

Imports and Initial Setup: The application imports necessary libraries like Flask, pandas, random, and get_sentiment from a custom sentimental module. It initializes the Flask app and loads datasets from CSV files (googleplaystore.csv and applestore.csv) and feedback from text files.

Data Preparation Functions:

  • get_top_playstore_apps(): Retrieves the top 10 Google Play Store apps by review count.
  • get_top_applestore_apps(): Retrieves the top 10 Apple App Store apps by rating count.

Routes:

  • / (Index Route): Handles both GET and POST requests. When POST is received, it extracts app_name and store from the form. Depending on the store selected, it filters the relevant dataset and retrieves app information. If the store is both, it retrieves data from both stores and includes additional comparison data. Returns the rendered index.html with the retrieved data and potential errors.
  • /run-analysis (Analysis Route): Handles GET and POST requests. Extracts selected fields for Google and Apple stores from the form. Calls a function from the analysis module to run the analysis and prepares image paths for the results. Returns the rendered analysis.html with images for the analysis results.

Error Handling and Feedback: If an app is not found in the selected store(s), an error message is set and displayed in the HTML template. Random feedback is selected from the loaded feedback and analyzed for sentiment.

HTML Page Inference (index.html):

  • Header and Navigation: Contains a header with the application title and a datetime display. Navigation links to Home, Run Analysis, and About Us pages.
  • Form for App Search: Provides input fields for app name and store selection (Google Play Store, Apple App Store, or Both). A submit button to send the form data to the Flask app.
  • Results Section: Displays error messages or app information depending on the search results. Shows app details from the selected store(s) and includes random feedback and sentiment analysis. If both stores are compared, it displays bar charts for top apps by review count (Google Play Store) and rating count (Apple App Store).
  • Charts: Uses Chart.js to render bar charts for the top 10 apps from both stores. Chart configuration ensures grids are hidden, and bars have light colors.
  • JavaScript: Updates the current date and time every second. Configures Chart.js to display data for both Google Play Store and Apple App Store charts based on the provided data.

General Observations: The Flask app efficiently handles data retrieval, sentiment analysis, and dynamic content rendering based on user input. The HTML is well-structured, incorporating modern design elements such as a clean layout, responsive design, and interactive charts. The combination of Flask routes and Chart.js provides a comprehensive and user-friendly interface for app comparison and analysis.
